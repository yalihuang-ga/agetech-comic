# **無障礙標準與高齡友善設計規格書 v1.0**

本規格書對接國際無障礙設計最低要求（WCAG 2.1 與歐盟 EN 301 549），並輔以高齡特定建議標準（SS 618:2016 與 ISO/TR 22411），確保產品具備極高的在地可行性與成效。

## **一、 核心無障礙基礎 (WCAG 2.1 & EN 301 549\)**

### **1\. 替代文字與非視覺內容 (Alt Text)**

* **規範對應**：WCAG 1.1.1 非文字內容 (A級)。  
* **前提（2026-07-21 前後端定案）**：漫畫採**單張四格圖**策略——一張圖內含 2×2 四格（利於 LINE 以單圖分享），`ComicResult.panels` 正常僅 1 個元素。  
* **實作規格**：  
  * 該單張 \<img\> 的 alt 為**依閱讀順序（左上→右上→左下→右下）的短版總述**（例如 alt="阿公的AI故事四格漫畫：早晨心情高興、上菜市場、和老伴散步、滿足回家"）。alt 不塞完整口述——螢幕閱讀器朗讀 alt 時無法暫停與逐段導航，過長反成負擔。  
  * **完整口述影像由劇場頁「完整故事」文字段落承擔**（後端 `narration` ＋前端 segments）：此為報讀器使用者的主要閱讀動線，可逐段導航、可暫停、與 TTS 朗讀同步。  
  * 朗讀進行中的**逐格視覺提示**以「象限高亮框」實作：朗讀至第 N 段時，於單張圖對應象限疊加高對比外框（沿用 segments 的 panelIndex 對位；不採 CSS 裁切法，避免 AI 格線誤差切壞畫面）。  
* **應用場景**：當患有重度視覺障礙的長輩開啟 iOS VoiceOver 或 Android TalkBack 等螢幕閱讀器時，系統能精準報讀出漫畫情節，確保資訊獲取絕對平權。

### **2\. 螢幕閱讀器輔助與焦點控制 (Screen Reader & Focus)**

* **規範對應**：WCAG 2.4.3 焦點順序 (A級)、WCAG 3.3.2 標籤或說明 (A級)。  
* **實作規格**：  
  * 網頁需使用正確的 HTML 語意化標籤（如 \<main\>, \<button\>），確保螢幕閱讀器能依照「先聽引導 $\\rightarrow$ 再聽漫畫 $\\rightarrow$ 最後聽分享按鈕」的正確邏輯順序朗讀。  
  * 所有圖示型互動按鈕（如麥克風、相機、大喇叭）必須加上清晰的 aria-label 說明，例如 \<button aria-label="按住這裡，用講的告訴我今天發生的事"\>。

### **3\. 色彩與對比度要求**

* **規範對應**：WCAG 1.4.3 對比度 (AA級)。  
* **實作規格**：網頁介面、按鈕以及漫畫圖內若有後製疊加的文字，其與背景的對比度必須嚴格大於 4.5:1。絕對不單純依賴「顏色」來傳遞資訊（例如不只用紅綠色區分對錯，需附帶清晰的打勾/打叉圖示）。

## **二、 高齡特定視覺與認知優化 (SS 618:2016)**

### **1\. 具象化的 IP 陪伴與防呆引導**

* **規範對應**：降低操作焦慮與科技門檻，強化共享陪伴。  
* **實作規格**：2D 陪伴員形象設計為一隻具備親和力的**溫暖貓咪**。這隻貓咪不僅作為國/台語 TTS 朗讀時的視覺發聲者，更會在長輩不知所措（閒置超過 5 秒）時，以溫和的動態呼吸燈提示並引導操作（例如語音提示：「阿公，點一下這隻貓咪旁邊的大按鈕，我唸今天的漫畫給你聽喔！」）。

### **2\. 降低認知負荷的視覺風格**

* **規範對應**：SS 618:2016 認知負荷管理。  
* **實作規格**：在後端呼叫 AI 算圖的 Prompt 中，需嚴格鎖定視覺清晰、乾淨的風格。例如使用色彩明亮的**日系彩色少年漫畫風格**，或是具備強烈對比、粗框線、大色塊的 **Neubrutalism（新野獸主義）風格**。嚴格禁止過度複雜的光影、漸層或雜亂背景，讓長輩能一秒看懂畫面情境，減輕大腦解讀畫面的壓力。

  > **釐清（適用範圍）**：本條「日系少年漫畫風／Neubrutalism」**僅適用於漫畫圖片的 AI 算圖風格，且為二擇一的舉例、非硬性指定**；不規範前端 App 的介面設計風格。真正的硬性要求是「視覺清晰、乾淨、強對比、禁止雜亂背景」。前端 UI 可自由選擇設計風格，只要符合本規格書其餘無障礙硬指標（對比 7:1、字級、觸控 ≥60px、Focus 外框等）。

### **3\. 多感官回饋與聽力退化補償**

* **規範對應**：SS 618:2016 聽力退化補償機制。  
* **實作規格**：  
  * 長輩點擊任何按鈕時，需同時提供「視覺（按鈕明顯下壓或變色）」與「聽覺（溫和的提示音）」雙重回饋。  
  * TTS 朗讀必須提供明顯的「語速調節」功能（1.0x / 0.8x / 0.6x），確保聽力退化或反應較慢的長者能以最舒服的節奏享受內容。
