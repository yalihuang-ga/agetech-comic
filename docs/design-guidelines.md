# 前端視覺與無障礙用色規範 v1.0

本文件是 `frontend/` 的**全站視覺與用色治理規範**，目的：在滿足無障礙硬指標（WCAG 2.1 / EN 301 549 / SS 618:2016）的前提下，讓整個 App 的畫面**一致、克制、好懂**，避免各頁各自加色造成長輩認知負荷。

- 適用範圍：所有前端頁面與元件。
- 對比標準：一律取較嚴的 **7:1（AAA）**（不採用 v1.md 提到的 4.5:1）。
- 對應實作：色彩/字級/間距的實際值集中在 [`frontend/src/app/globals.css`](../frontend/src/app/globals.css) 的 CSS 變數；本文件規範「怎麼用」。

> 高齡無障礙的視覺原則常與一般 App/網站的直覺相反：首頁不是 dashboard 而是啟動器；顏色是功能不是裝飾；可預測、可記憶 > 新奇。

---

## 一、用色治理原則（最重要）

1. **中性為底**：頁面、卡片、多數按鈕都用中性（碳灰字 ＋ 象牙白底）。
2. **一頁一重點色**：每個畫面**只有一個主行動**使用重點色（磚紅 Terracotta）；其餘一律中性描邊。
3. **語意色只用於語意**：綠＝完成、紅＝錯誤、黃＝提醒，且**一定同時搭配文字＋圖示**，不可只靠顏色。
4. **對比 ≥ 7:1（AAA）**：所有文字、圖示對其底色。
5. **顏色不是唯一資訊（WCAG 1.4.1）**：選中、對錯、狀態，都要有文字或圖示佐證。
6. **克制**：一個畫面最多 1 個重點色 ＋（必要時）1 個語意色，不三四色並陳。
7. **中明度色不承載文字**：亮磚紅、芥末等中間調配黑字或白字都到不了 7:1，只能當裝飾（描邊、插圖、圖形）；要放字一律用深階磚紅（白字）或「淡底＋深字」。
8. **一色一語意**：磚紅＝互動語意專用（主行動、選中狀態、阿咪品牌）；芥末黃＝插圖層專用（模式卡 icon 填色）。兩者不得互換，避免「插圖看起來像被選中」的誤讀。

## 二、角色化色票

暖色調定案版（2026-07，經 Adobe/WebAIM 驗證）。對比為 WCAG 公式**精確值**，前景色對其底色皆 ≥ 7:1（AAA）。

| 角色 | 用途 | 色值 | 對比（精確） |
|---|---|---|---|
| 淺象牙白 | 頁面背景（降炫光、白內障友善） | `#FAF6EE` | — |
| 白 | 卡片 / 表面（描邊、不填色） | `#FFFFFF` | 中性 |
| 碳灰 | 正文 | `#2E2E2B` on `#FAF6EE` | **12.64:1** |
| 淺碳灰 | 次要文字（**下限 `#4D4D48`**，再淺跌破 7:1） | `#4A4A45` | **8.27:1** |
| **磚紅 Terracotta** | **主行動**（白字）／主題 icon 填色 | `#8C3B24` | 白字 **7.59:1**；對 bg 7.04:1 |
| 亮磚紅 | ⚠️ 僅裝飾 / hover，**不承載文字**（白字僅 6.0:1） | `#A0492C` | — |
| 深咖啡 | 次要動作（白字），不與磚紅爭主 | `#3E2C23` | 白字 **13.2:1**；對 bg 12.26:1 |
| 復古芥末黃 | **插圖層專用**（模式卡 icon 填色；靠深描邊 `#1E1E24` 定形，**不承載文字**） | `#D4AC2F` | 深描邊對其 7.9:1 |
| 深綠 | 語意・完成（配 ✓＋文字） | `#05582A`（白字） | ≥ 8:1 |
| 深紅 | 語意・錯誤（配 ⚠＋文字） | `#8A1C1C`（白字） | ≥ 8:1 |
| 暖黃底 | 提醒（碳灰字；已加深以免融入象牙白底） | `#F5E3A8` | 10.7:1 |
| 純黑底 | 字幕（白 / 亮黃字；規格指定，不隨主題變） | `#000` + `#FFF`/`#FFFF00` | 21:1 / 19.6:1 |
| 紫 | Focus 外框（4px，絕不 `none`；非文字 ≥3:1） | `#7A00CC` | 明顯 |

**重點色只有一個**：磚紅代表「去做」。深咖啡是次要，不與磚紅在同一畫面爭主。想要「亮磚紅感」的合規替代：磚紅淡底 `#F7DED4` ＋ 磚紅深字 `#7A2E17`（≈8:1），或亮磚紅只做描邊/插圖。

## 三、字級與行高（SS 618:2016）

| 項目 | 規範 |
|---|---|
| 正文 | ≥ **20px** |
| 按鈕 / 字幕 | ≥ **24px** |
| 標題 | **28–40px** |
| 行高 | **1.5–1.6** |
| 段距 | ≥ 字級 **1.5 倍** |

---

## 四、觸控與間距

| 項目 | 規範 |
|---|---|
| 主要控制觸控目標 | ≥ **60px**（本專案採 64px） |
| 主要控制之間的間距 | ≥ **24px** |
| 輔助控制（如移除鈕、卡內 tag） | ≥ **44px**（WCAG 2.5.5） |
| 輔助控制之間的間距 | ≥ **16px**（44px 級小控制；主要控制仍用 24px） |

---

## 五、焦點與鍵盤

- `:focus-visible` 給 **4px 高對比紫色外框**，**嚴禁 `outline:none`**。
- 焦點順序需符合邏輯（左至右、上至下）。
- 路由切換後焦點自動移到該頁 `<h1>`；提供 skip-link；使用 `<main>`/`<nav>` 語意地標。
- 不可產生鍵盤陷阱。

## 六、圖示與狀態

- **文字為主、圖示為輔**：任何互動一定有文字標籤，不做 icon-only。
- **Icon 採二層制**：
  1. **功能 icon**（返回/設定/播放/勾叉/箭頭…）：單色線條、粗筆畫、`currentColor`——自動繼承按鈕文字顏色，對比永遠與文字一致（≥7:1）。實作：`frontend/src/components/Icon.tsx`（零相依、inline SVG）。
  2. **插圖 icon**（相機/麥克風/點選手勢/阿咪）：彩色粗描邊風格，只用於模式選擇卡與品牌位置，**一頁最多一組**。檔案在 `public/assets/icons/`、`public/assets/characters/`。
- **功能按鈕禁止使用 emoji**（顏色不受控、跨平台不一致、稚氣）；**內容圖卡**（心情/地點/事件選項）的 emoji 屬內容插圖，暫予保留、日後換自製 pictogram。
- **例外（PM 定案）**：tag chip（篩選按鈕）上的 emoji 屬「類別辨識圖」——雖長在互動元件上，但其功能是幫長輩辨識類別（與選項卡同性質），准予保留；其餘互動元件不適用此例外。
- **陪伴員阿咪＝會呼吸的貓臉插圖**（`components/AmiCat.tsx` 之 inline SVG：橘毛 `#F4A261`、描邊 `#1E1E24`、內耳芥末黃、磚紅鼻）。動畫規則：呼吸/眨眼幅度小、`prefers-reduced-motion` 自動停止、**動畫不承載資訊**；閒置 10 秒＝呼吸加大＋文字提示泡（**無自動語音、任何互動即消失**）。旁有文字標籤時貓臉一律 `aria-hidden`。
- **狀態一律「圖示＋文字」**：完成＝✓＋「今天記好了」、錯誤＝⚠＋白話說明；不可只用顏色區分。

## 七、元件慣例

- **主行動**：填色重點色大按鈕，**一頁一顆**。
- **次要動作**：中性描邊按鈕，或清單列（右側 chevron 表示「裡面還有」）。
- **卡片**：白底 ＋ 描邊，**不用彩色底**。
- **錯誤 / 提醒**：語意色框 ＋ 圖示 ＋ 白話文字（**禁止**出現 Error code）。
- **首頁**：是啟動器不是 dashboard——聚焦一個主行動，其餘安靜；統計/鼓勵移到對應頁面。

### 元件庫（實作對應 `frontend/src/components/`）

| 元件 | 規格 |
|---|---|
| **TagChip**（篩選 chip） | 圓角膠囊、min-height **48px**、描邊 2px、chips 間距 **24px**；單選；選中＝磚紅底白字＋**✓**（不靠純色彩）；`aria-pressed`；chip 上 emoji 為類別辨識圖（准用例外） |
| **PawCard**（集點卡） | 取代細進度條：7 大格（實心貓掌章／虛線空格）、里程碑格掛 gift 線條 icon；進度**同時以文字說明**；每格附 `sr-only` 狀態文字 |
| **StampCard**（媒體卡片） | 白底描邊卡；**cover 4:3 撐滿卡寬**（未有正式圖用「準備中」佔位圖，alt 註明）；下方依序 title(h3, 22px)、日期(磚紅)、tag 列（輔助控制 44px、間距 16px） |
| **Disclosure**（揭示區） | 取代 hover tooltip（觸控無 hover、WCAG 1.4.13、手抖誤觸）：大按鈕（≥56px）＋ `aria-expanded`/`aria-controls`，內容 inline 展開；規則類文字**從參數動態生成**，避免文案與設定脫鉤 |

## 八、Do / Don't

**Do**
- 中性為底、一頁一重點色。
- 狀態配文字＋圖示。
- 對比 ≥ 7:1、觸控 ≥ 60px、字 ≥ 20px。
- 同功能同位置同外觀（可預測、可記憶）。

**Don't**
- 多色卡片並陳、一頁三四種重點色。
- icon-only、滿版 emoji。
- 純用顏色區分對錯 / 選中 / 狀態。
- `outline:none`、字 <20px、觸控 <60px。
- 首頁塞 dashboard（多統計、多入口並列）。
