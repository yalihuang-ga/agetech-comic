# **產品原型規格書：【樂齡時光繪本】AI 跨世代記憶共創平台**

## **一、 專案定位與開發背景**

* **競賽組別**：高齡科技產業-樂齡數位共創推動計畫「友善生活組」（聚焦社會參與、智慧生活與心理陪伴）。  
* **核心痛點**：台灣 60 至 70 歲長者雖有 80.2% 的智慧型手機持有率，但真正會操作 App 的比例僅 41.8% ；且近 50% 的長者因恐懼詐騙而拒絕複雜的個資輸入 。長者手部機能退化，常導致精準觸控困難 。  
* **核心動能**：調查顯示，促使未上網長者願意嘗試科技的最強烈動能是「與子女、朋友聯繫」。  
* **解決方案**：本平台將複雜的 AI 漫畫生成流程與虛擬人技術進行「減法優化」。透過 LINE 一鍵免密登入與導引式大圖示圖卡，讓長者用點選方式即可將生命故事生成「四格漫畫」，並由「靜態表情包虛擬說書人」以慢速台/國語朗讀，最後一鍵分享至 LINE 家族群組 。

## **二、 使用者角色與預設資產（靜態立繪表情包）**

為控制算力成本並確保視覺穩定度，系統不即時生成人物模型，改採預製的靜態去背立繪（Visual Novel 風格），並根據劇情文字切換圖片路徑：

| 角色名稱 | 形象特徵 | 適用情境/性格 |
| :---- | :---- | :---- |
| **阿公** | 戴遮陽斗笠、脖子掛運動毛巾、Polo 衫。 | 適合中南部傳統農村、喜歡種菜散步的阿公。 |
| **阿嬤** | 俐落捲髮、老花眼鏡、鮮豔針織衫。 | 適合喜歡去樂齡中心上課、與閨蜜聊天的活躍長輩 。 |
| **小孫子/小孫女** | 朝氣十足的二次元年輕角色、運動服。 | 數位志工形象，作為幫長輩溫暖說故事的說書人 。 |

## **三、 功能模組與前端流程規劃 (User Flow)**

1\. 登入階段：LINE 一鍵免密登入（滿足註冊績效指標 ）

* **介面**：極簡畫面，中央放置一個超大綠色按鈕「用 LINE 開始建立我的回憶錄」。  
* **邏輯**：點擊後模擬 LINE Login 授權，成功後於後台建立或綁定使用者識別碼（User ID），直接跳轉至主畫面。拒絕傳統 Email、密碼等繁瑣且易引發資安焦慮的輸入 。

2\. 輸入階段：導引式故事圖卡（取代文字打字，降低認知負荷 ）

* **介面**：每次畫面僅出現 1 個問題與 3\~4 個「超大尺寸（大於 60px）」圖示按鈕 。  
* **互動流程範例**：  
  * *問題一*：「阿公/阿嬤，今天心情怎麼樣？」➔ 按鈕：\[高興\]、\[平靜\]、\[有點累\]。  
  * *問題二*：「今天去了哪裡呀？」➔ 按鈕：\[菜市場\]、\[公園散步\]、\[樂齡中心\]、\[待在家裡\] 。  
  * *問題三*：「有沒有發生什麼有趣的事？」➔ 按鈕：\[跟孫子講電話\]、\[買到好吃的\]、\[跟朋友泡茶\] 。  
* **技術邏輯**：前端將長者點選的 Key-Value 組合（例如：高興 \+ 樂齡中心 \+ 跟朋友泡茶），自動包裝成一段結構化 Logline（故事大綱短語），發送至後台 AI 漫畫生成 Flow 。

### **3\. 展示階段：虛擬人劇場 ＋ 四格漫畫生成**

* **介面排版（響應式舞台分離法）**：  
  * **左側/上方區塊**：展示所選角色的「靜態去背立繪表情包」，下方為「高對比不透明底框字幕區（純黑底、純白字，字體至少 24px）」，確保視覺清晰度 。  
  * **右側/下方區塊**：展示 AI 生成的四格漫畫（或條漫）成品 。  
* **互動邏輯**：  
  * 播放語音 TTS（模擬調慢至 0.6x \~ 0.7x 的國/台語慢速語音）。  
  * 當語音播到特定劇情（如：中獎、高興），前端使用 JS 切換左側 \<img src="..."\> 的路徑（例如從 smile.png 切換至 laugh.png），達成電子小說式的生動感。

### **4\. 輸出與留存階段：一鍵分享與回憶存摺**

* **介面**：漫畫下方提供兩個醒目按鈕：  
  * 按鈕 A（綠色）：\[傳送給我的孫子（分享到 LINE）\] ➔ 觸發 LINE 分享 API，將「漫畫網址/圖片」與客製化溫暖長輩問候語一鍵發送至家族群組 。  
  * 按鈕 B（藍色）：\[打開我的回憶存摺（歷史紀錄）\] ➔ 進入歷史清單頁面，可瀏覽過去每一天生成的漫畫，建立數位成就感與持續使用意願。
