# **樂齡時光繪本】國際無障礙與高齡人因指標前端開發合規規格書**

本文件旨在將計畫要求的最低無障礙規範（WCAG 2.1 / 歐盟 EN 301 549） ，以及高齡特定建議標準（新加坡 SS 618:2016、ISO/IEC Guide 71、ISO/TR 22411 人因工程框架） ，轉化為前端網頁原型（Web Prototype）的具體代碼實作指南，用以補足長者在視覺、聽覺、操作機能、認知負荷與使用習慣上的特殊設計需求 。

## **基準層：數位服務可近用性最低要求**

對應標準：WCAG 2.1 (AA級) / 歐盟 EN 301 549 資訊通訊技術無障礙標準 核心目標：確保系統在底層程式碼結構上，能被各類輔助技術（如螢幕閱讀器）正確解析 。

### **1\. 語意化標籤與鍵盤可操作性 (Operable)**

* **代碼規範**：所有可點擊的故事圖卡、登入按鈕、分享鍵，必須使用標準的 \<button\> 或 \<a\> 標籤，嚴禁使用 \<div\> 或 \<span\> 綁定 click 事件。  
* **鍵盤聚焦**：長者或障礙者使用鍵盤 Tab 鍵切換時，被選中元件必須呈現粗黑、高對比的「聚焦外框（Focus Outline）」（CSS 不得寫 outline: none）。  
* **焦點順序**：頁面的 Tab 導覽順序必須符合邏輯（左至右、上至下），點擊「LINE 登入」或進入故事生成頁時，焦點必須自動移至當前主要互動區域，不可產生「鍵盤陷阱」。

### **2\. 全感官輔助與資訊可感知性 (Perceivable)**

* **替代文字**：產出的四格漫畫為**單張 2×2 四格圖**（前後端 2026-07-21 定案），該 \<img\> 必須動態綁定 alt 屬性，內容為依閱讀順序的短版總述（例如：alt="阿公的AI故事四格漫畫：早晨起床、上菜市場、遇見老友、滿足回家"）；完整口述影像由劇場「完整故事」文字段落承擔（詳見 v1 規格書 §一-1）。  
* **動態更新公告 (ARIA Live Zones)**：  
  * 當 AI 正在生成漫畫、或者虛擬說書人的對話字幕更新時，該字幕容器必須加上 aria-live="polite" 與 aria-atomic="true" 屬性。  
  * 確保有視覺障礙或使用螢幕閱讀器的長者，能即時接收到系統狀態的語音播報。

## **優化層：高齡特定視覺與操作細節**

對應標準：SS 618:2016 新加坡銀髮產業產品設計標準 核心目標：針對高齡者常見的生理機能衰退（如白內障、重度老花、手部肌肉細微機能退化與觸控不靈敏），優化介面物理快適性 。

1\. 視覺適配性（防老花與色彩干擾）

* **文字規範**：正文字體大小預設不得低於 **20px**，對話框與按鈕文字建議 **24px 以上**，標題 **28px 以上**。行高（Line-height）至少設定為 **1.5倍**，段落間距至少為字體大小的 **1.5倍**。  
* **色彩對比**：操作性文字、圖示按鈕、字幕與其底色，對比度必須超越 WCAG 規定的 4.5:1，一律提升至 **7:1（AAA級標準）**。系統對話框底色採用純黑（\#000000）搭配純白（\#FFFFFF）或亮黃（\#FFFF00）文字。  
* **禁止色彩依賴**：不可單純用顏色區分錯誤或成功（例如：紅鈕代表錯誤、綠鈕代表成功），必須同時搭配清晰的**大文字提示**或**打勾/打叉圖標**。

2\. 物理操作防錯機制（防手抖與精準度退化）

* **觸控目標體積**：  
  * 長者指尖觸控敏感度降低，且易受手部機能障礙影響 。  
  * 所有可點擊的故事選擇圖卡，其有效點擊區域（Click Target）寬高必須大於 **60px × 60px**，且按鈕與按鈕之間的物理間距（Margin）必須大於 **24px**，防止長者誤觸鄰近按鈕 。  
* **防重複點擊防禦 (Debouncing)**：長者因手指不自覺顫抖，容易連續快點兩下按鈕導致系統出錯 。  
  * 前端 JavaScript 必須對所有提交按鈕實作「防抖動機制（Debounce）」，限制在 **1.5 秒內**的重複點擊視為單次有效，或在點擊後立即將按鈕設定為 disabled。

## **減法層：高齡認知負荷與習慣調適**

對應標準：ISO/IEC Guide 71 & ISO/TR 22411 高齡者與身心障礙者需求之人因工程框架 核心目標：對抗高齡長者在「減法階段」易產生的科技焦慮、不安全感、以及記憶力/認知速度下降等障礙 。

1\. 消除科技不安感與界面隱形化

* **導引式流程（線性引導）**：畫面拒絕呈現複雜的多功能選單（如傳統網站的 Header / Sidebar / 複雜設定區）。採用「一頁一問題（One Thing Per Page）」的線性步驟流程（LINE登入 ➔ 選擇心情 ➔ 選擇地點 ➔ 觀看說書劇場），絕不跳出擾人的彈出式視窗（Modal Popups）或蓋台廣告，避免長者迷失操作方向。  
* **自然語音對話支援**：系統語音播放速度（TTS）預設調慢至 **0.6倍至 0.7倍速** 。播放期間，畫面上配置大大的「暫停/繼續」按鈕，允許聽覺與認知速度較慢的長者反覆聆聽，不設定任何會造成心理壓力的倒數計時器。

### **2\. 人因防禦機制（防錯與容錯）**

* **無痛容錯設計**：長者極度害怕「按錯按鈕會弄壞設備、導致財產損失或系統當機」 。  
  * 介面上提供隨時可點擊的「返回上一步」大按鈕。  
  * 錯誤訊息提示嚴禁出現任何代碼（如 Error 500、NullPointer），一律轉化為具溫度的在地化白話指引，例如：「阿公，網路上傳好像有點累，請幫我點右邊的黃色按鈕再試一次喔！」。
